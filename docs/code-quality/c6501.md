# <a name="c6501"></a>C6501
警告 C6501： 註釋衝突：\<名稱 > 屬性與先前指定的屬性發生衝突  
  
> [!NOTE]
>  只有在使用原始程式碼註釋語言 (SAL) 已取代的版本的程式碼中，會發生這個警告。 我們建議您將移植程式碼以使用最新版的 SAL。 如需詳細資訊，請參閱[使用 SAL 註釋減少 C/c + + 程式碼缺失](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md)。  
  
 這個警告表示註解中有衝突的屬性存在。 這通常會有類似用途的多個屬性用來標註的參數或傳回值時發生。 若要修正警告，您必須選擇最符合您需要的屬性。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告，因為 ValidElementsConst 和 ValidBytesConst 提供一個機制，允許有效要讀取的資料：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void fd([SA_Pre(ValidElementsConst =4, ValidBytesConst =4)] char pch[]);  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f( [Pre(ValidElementsConst=4, ValidBytesConst=4 )] char pch[] );  
```  
  
 若要修正這個警告，請使用最適當的屬性，如下列程式碼所示：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f( [SA_Pre(ValidElementsConst=4)] char pch[] );  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f( [Pre(ValidElementsConst=4)] char pch[] );  
```
